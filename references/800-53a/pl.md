# NIST SP 800-53A Rev 5 — PL: Planning Assessment Procedures

Assessment procedures for the Planning family evaluate whether the organization develops, documents, and maintains system security plans, rules of behavior, and other planning artifacts.

---

## PL-1: Policy and Procedures

**Assessment Objective:** Determine if the organization develops, documents, disseminates, reviews, and updates planning policy and procedures.

**Examine:** Planning policy; planning procedures; system security plan.

**Interview:** Organizational personnel responsible for planning policy.

**Test:** Organizational processes for managing planning policy.

**Determination Statements:**
- PL-01a. — A planning policy is developed, documented, and disseminated
- PL-01b. — Planning procedures are developed, documented, and disseminated
- PL-01c. — The policy and procedures are reviewed and updated at an organization-defined frequency

---

## PL-2: System Security and Privacy Plans

**Assessment Objective:** Determine if the organization develops, documents, reviews, and updates system security and privacy plans.

**Examine:** Planning policy; system security plan; privacy plan; system design documentation; organizational risk management documentation.

**Interview:** Organizational personnel responsible for security and privacy planning; system owners; information system security officers.

**Test:** System security plan review and update processes; plan approval mechanisms.

**Determination Statements:**
- PL-02a.[01] — A system security plan is developed that defines the authorization boundary of the system
- PL-02a.[02] — The plan describes the operational context of the system in terms of mission and business processes
- PL-02a.[03] — The plan provides the security categorization of the system including supporting rationale
- PL-02a.[04] — The plan describes the operational environment for the system and any dependencies or connections to other systems
- PL-02a.[05] — The plan provides an overview of the security requirements for the system
- PL-02a.[06] — The plan identifies any relevant overlays
- PL-02a.[07] — The plan describes the security controls in place or planned for meeting the security requirements
- PL-02b. — The system security plan is distributed to organization-defined personnel or roles
- PL-02c. — The system security plan is reviewed at an organization-defined frequency
- PL-02d. — The system security plan is updated to address changes to the system, environment of operation, or problems identified during plan implementation or control assessments
- PL-02e. — The system security plan is protected from unauthorized disclosure and modification

---

## PL-4: Rules of Behavior

**Assessment Objective:** Determine if the organization establishes and provides rules describing responsibilities and expected behavior for system use.

**Examine:** Planning policy; rules of behavior; signed acknowledgments; system security plan.

**Interview:** Organizational personnel responsible for establishing rules of behavior; system users who have signed rules of behavior.

**Test:** Rules of behavior distribution and acknowledgment processes.

**Determination Statements:**
- PL-04a. — Rules are established and made available that describe the responsibilities and expected behavior for system and information use
- PL-04b. — A signed acknowledgment is received from individuals indicating that they have read, understand, and agree to abide by the rules of behavior before authorizing access
- PL-04c. — The rules of behavior are reviewed and updated at an organization-defined frequency
- PL-04d. — Individuals who have signed a previous version of the rules of behavior are required to read and re-sign when the rules are revised/updated
